・テストバージョン作成

----------------------------------------------------
----------------------------------------------------

・モジュールを作成する場合は、ファイルが格納されているディレクトリをカレントにして、
    cvs import -m "コメント" 格納先ディレクトリ ベンダータグ リリースタグ
  とします。
  加えなくないファイルやディレクトリを指定するには -I <file> とします。

・リポジトリからファイルを取り出すには、
    cvs checkout モジュール名
  とします。
  実行すると、カレントディレクトリに「モジュール名」のディレクトリが作成され、ファイルが格納されます。

・リポジトリを更新するには、
    cvs commit -m "コメント" <ファイル名 ...>
  とします。ファイル名を指定する事で、特定のファイルを対象にします。

・リポジトリにファイルを追加するには、
    cvs add ファイル名 ...
  とします。

・リポジトリのコピーを削除する場合は、ファイルが格納されているディレクトリの一つ上のディレクトリで
    cvs release -d モジュール名
  とします。

・作業中のファイルを、リポジトリに格納されたファイルと一致させる為には、
    cvs update <ファイル名...>
  とします。
  ※ディレクトリを再帰的にチェックするには、
    cvs update -R -d
    とします(本当は、cvs checkout なんでしょうか)。

・作業中のファイルが、最新かどうか確認するには、
    cvs status <ファイル名...>
  とします。

・作業中のファイルと、リポジトリに格納されたファイルとの差分を調べるには、
    cvs diff -r <リポジトリ> <ファイル名...>
  とします。

・ファイルを削除する場合は、
    cvs remove -f <ファイル名...>
  とします。-f オプションで、作業中のファイルを直ちに削除します。

・格納されているファイルの一覧は、
    cvs log -R
  とします。

・別のリポジトリからファイルを取ってくる場合は、
    cvs -d リポジトリのある場所 command...
  とします。
  例えば、環境変数 CVS_DIR なんかを設定して、
    cvs -d $CVS_DIR checkout hoge
  などとするといいでしょう(win では cvs -d %CVS_DIR% checkout hoge)。


オプション説明

 cvs <options> command ...

 options に指定できるもの

 -n	チェックだけ行い、ファイルの更新を行わない。
 -q	冗長なコマンドの出力が抑止される。
 -H	コマンドの説明を表示(続けてコマンドも指定する)


環境設定

・CVS の環境設定を行う場合、作業ディレクトリにて
    cvs checkout CVSROOT
  とします。
  実行すると、環境設定ファイルが作業ディレクトリにコピーされます。

・cvsignore というファイルで、デフォルトで無視するファイルを指定します。現在いくつか追加してあります。
・modules というファイルで、モジュールの別名を指定します。これも現在いくつか指定してあります。
  ※

・編集した環境設定ファイルを更新するときは、
    cvs commit
  とします。これで、安全に CVS の環境設定を行う事ができます。
